Death Stranding Terrain Scan System

Custom terrain pulse post-process effect with Fresnel bitmask highlighting, a first-person visor mode, and live object distance tracking.

Genre
Standalone Game System
Platform
PC (Windows)
Engine
Unreal Engine 5.2
Tools
C++, Blueprint Scripting

Project Goal

Recreate and expand upon the scan pulse mechanic from Death Stranding as a standalone, modular gameplay system — enhancing player interaction with the game world through detection, visual highlighting, and information overlays.

Core Features

Terrain Scan Pulse

Forward-emitting pulse that visually scans the environment, highlighting terrain, buildings, and structures in the player's line of sight.

Scannable Object Interface

Unified interface implemented by all scannable actors for modular, scalable detection across any actor type.

Objective Detection

Detection and tagging of structures linked to current objectives, with visual indicators to guide player progression.

Threat Detection

Identifies hidden dangers such as underground mines, displaying warning markers to inform the player of nearby threats.

First-Person Visor Mode

Zoomable visor overlaying detailed object information, driven by a per-frame line trace from the camera.

Pinning & Distance Tracking

Pin any scanned object to show a persistent on-screen marker with real-time distance from the player.

Post Process Material Breakdown

1. Scan Texture

Combines a grid and gradient texture projected in world space via WorldAlignedTexture — keeping the effect seamless across any surface. SceneTexture WorldNormal ensures correct projection over terrain geometry.

  • GridSize — Adjusts density of grid lines across world space
  • GradientSize — Controls the width of the gradient band / pulse edge

2. Sphere Mask (Animated Pulse)

A hollow sphere mask centered on the player, radius animated by a Blueprint timeline to produce the expanding scan pulse.

  • PlayerPosition — Continuously set via Material Parameter Collection (MPC)
  • Radius — Animated by timeline for expanding scan
  • Hardness — Controls softness/thickness of the ring's edge

3. Fresnel Highlighting with Bit Masking

Dynamic object highlighting using a Fresnel effect, with highlight colour determined by each object's stencil value via bit masking. Allows multiple highlight colours to coexist based on object-specific stencil values (e.g. threats vs. objectives).

ScanObject Interface

Applied to all actors that can be scanned via the visor. A per-frame line trace checks if the hit actor implements the interface, then calls the appropriate function.

  • setIsThreat(bool) — Marks the actor as a threat; appearance reacts accordingly
  • getIsThreat() — Returns threat status, used by UI and stencil assignment at BeginPlay
  • pin() / unpin() — Enables/disables persistent distance tracking for the actor
  • focus() — Called on visor hover; triggers info widget display
