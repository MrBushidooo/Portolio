Project Goal

Design and implement a multiplayer FPS using Unreal Engine 5 with a dedicated server architecture — building a network-aware, server-authoritative experience similar to real online competitive shooters.

Key learning objectives:

  • Multiplayer architecture in Unreal Engine (Client-Server model)
  • Server-authoritative gameplay design, replication, RPCs, and network ownership
  • Dedicated server deployment and testing with AWS GameLift
  • Session management (Create/Join flow) and multi-client testing

Project Overview

A server-authoritative multiplayer FPS in UE5.5, demonstrating networking architecture and competitive combat mechanics. All core gameplay logic — firing, hit detection, damage — is validated on the server to maintain fairness and synchronisation across clients.

The project integrates AWS GameLift for cloud dedicated server hosting with a session-based matchmaking flow, simulating a real production multiplayer pipeline.

Core Features & Systems

Movement System

Replicated character movement with server-validated sprint and client-server synchronisation across all connected players.

Weapon System

Server-authoritative line-trace shooting with replicated muzzle flash, impact VFX, and sound. Damage confirmed server-side via RPC.

Aim Down Sights

Hold-based ADS with smooth camera FOV transition. Replicated ADS state ensures correct animation alignment for all clients.

Flash Grenade

Physics-simulated throwable, server-spawned and replicated. Screen-blind effect applied via Client RPCs after server-side distance checks.

Reveal Grenade

Reveals enemy positions within a radius via server-side radial scan. Wall-through visibility using post-process materials and custom depth.

Bullet Decals & Hit VFX

Server-authoritative hit confirmation spawns impact decals and VFX, replicated so all clients see consistent hit feedback.

Networking Architecture

Server-Authoritative Design

All core gameplay logic is validated on the server before being replicated to clients, ensuring fairness and preventing exploits.

RPCs

  • Server RPCs: Player input (firing, ADS, grenades) sent from client to server for validation
  • Client RPCs: Effects like flash blindness sent from server back to specific clients
  • Multicast RPCs: VFX events broadcast to all connected clients

AWS GameLift Integration

Server builds are deployed to a live cloud environment. A REST API integration handles client-side matchmaking to connect players to active dedicated servers, enabling remote testing in a real online infrastructure.
